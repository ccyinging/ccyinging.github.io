mysql安全管理(MariaDB 10.4.17 for Linux)

2019-11-17 13:05:21
  • 学习记录
  • mysql

​ mysql系统数据库记录了mysql的账户信息以及mysql账户的访问权限，进而实现mysql账户的身份认证以及权限验证，在mysql系统数据库mysql中：

  • user表：mysql账户的身份认证以及mysql服务实例级别账户的权限验证；
  • db表：mysql数据库级别账户的权限认证；
  • tables_priv表：mysql表级别账户的权限认证；
  • columns_priv表：mysql字段级别账户的权限认证；
  • process_priv表：mysql存储程序级别账户的权限认证；

注意事项，消除安全隐患

  • 不要给mysql账户赋予比其需要还要多的权限
  • 尽量使用grant、invoke、drop等命令来操作权限表，避免使用insert、update、delete等更新语句，更新语句可能会造成各级别权限表出现不一致的情况；使用更新语句操作表后，要用“flush privileges;”命令刷新权限
  • 删除匿名账户和登录主机可以是任何主机的账户
  • 修改root的账户名和密码
  • 修改mysql服务的端口号，为远程连接提供安全的账户名、密码以及连接主机
  • 在配置文件[mysqld]选项组中添加“local_infile=0”，禁止mysql读取读取客户机文件
  • 除非备份等必要工作，避免mysql账户授予file权限，防止在服务器上创建文件
  • 为mysql服务提供安全的防火墙
